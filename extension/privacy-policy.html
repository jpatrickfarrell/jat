Privacy Policy

JAT Bug Reporter Browser Extension — Last updated: January 30, 2026

Summary: JAT Bug Reporter stores all captured data locally on your device. Nothing is collected, transmitted, or shared unless you explicitly submit a bug report to your own configured server.

What This Extension Does

JAT Bug Reporter helps developers create detailed bug reports by capturing diagnostic information from web pages. It captures screenshots, console logs, network requests, and element information when you explicitly trigger each action.

Data Collected

When you use the extension, the following data may be captured at your request:

  • Screenshots — Images of the visible browser tab or selected elements. Captured only when you click the screenshot button.
  • Console logs — JavaScript console output (log, error, warn, info, debug, trace) from the active page. Sensitive values such as API keys, tokens, and passwords are automatically redacted before storage.
  • Network requests — HTTP request and response metadata (URLs, status codes, timings, sizes). Request/response bodies are not captured.
  • Element data — HTML tag name, CSS selectors, XPath, class names, and bounding dimensions of elements you select with the element picker.
  • Bug report text — Title, description, type, and priority that you enter into the bug report form.

How Data Is Stored

All captured data is stored locally on your device using the browser's built-in chrome.storage.local API. Data remains on your machine and is never sent anywhere automatically.

Storage limits per session:

  • Screenshots: up to 10
  • Console log entries: up to 200
  • Network requests: up to 50
  • Selected elements: up to 20

You can clear all captured data at any time using the clear buttons in the extension popup.

Optional Server Submission

If you configure a Supabase project URL and API key in the extension settings, you may submit bug reports to your own server. This is entirely optional and disabled by default.

When you submit a report:

  • The bug report text and attached screenshots are sent to the Supabase project you configured.
  • No data is sent to JAT developers or any third party.
  • You control the server, the database, and the data.

If no server is configured, bug reports are saved locally only.

Data We Do Not Collect

  • No personal information (name, email, location)
  • No browsing history or activity tracking
  • No analytics, telemetry, or usage statistics
  • No cookies or authentication tokens from visited sites
  • No data transmitted to JAT developers or third parties

Permissions Explained

  • activeTab — Access the current tab to capture screenshots and run the element picker. Only activates when you click the extension icon.
  • storage — Store captured data and your extension settings locally on your device.
  • scripting — Inject the content script that enables element picking and console log capture on the active page.
  • webRequest — Monitor network requests for the network capture feature. Used to record request metadata only.
  • contextMenus — Add a "Report Bug with JAT" option to the right-click menu.
  • host_permissions (<all_urls>) — Required for the content script and network monitoring to work on any website you visit. The extension does not read or modify page content unless you trigger a capture action.

Third-Party Services

The extension includes the Supabase JavaScript client library. This library is only activated if you configure your own Supabase credentials in the extension settings. No connection to any Supabase server is made until you explicitly submit a bug report.

Data Retention

Locally stored data persists until you clear it (via the extension UI or by uninstalling the extension). There is no automatic data expiration. If you submit reports to your own Supabase server, data retention on that server is governed by your own policies.

Children's Privacy

This extension is a developer tool and is not directed at children under 13. We do not knowingly collect data from children.

Changes to This Policy

Updates to this privacy policy will be reflected in the extension's version notes and on this page. The "Last updated" date at the top will be revised accordingly.
